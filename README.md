Структура проекта

Dockerfile : Dockerfile — это файл конфигурации для построения образа Docker. 
    Вы напишете содержимое этого файла на основе API.
cmd : cmdКаталог обычно содержит точку входа вашего приложения. 
    Подкаталог server предполагает, что сервер API является основным компонентом вашего проекта. 
    Файл main.go в cmd/serverсодержит точку входа приложения.
internal : internalКаталог помогает организовать внутренние пакеты вашего приложения. 
    Внутренние пакеты не должны экспортироваться, что делает их отличным местом для сокрытия деталей реализации.
    http : этот подкаталог будет содержать код, связанный с HTTP, 
    включая обработчики маршрутов и, возможно, промежуточное программное обеспечение для вашего API.
    handlers.go : в этот файл вы включите обработчики HTTP-запросов.
    users.go : Здесь вы указываете обработчики HTTP, связанные с управлением пользователями.
    models : этот каталог будет содержать код, связанный с базой данных, и модели данных.
    database.go : этот файл будет содержать код для инициализации и подключения к базе данных.
    migrations.go : этот файл управляет миграциями схемы базы данных, гарантируя, 
    что схема вашей базы данных соответствует требованиям вашего приложения.
users.go : этот файл будет содержать модели данных или определения структур, 
    связанные с пользователями, взаимодействующими с базой данных.
    пользователи : этот каталог будет содержать логику, специфичную для пользователя.
    user.go : Этот файл включает функции и структуры, связанные с управлением пользователями, 
    которые взаимодействуют с реализацией базы данных. Реализация HTTP взаимодействует с функциями здесь.

Создаем докер файл
Первая часть    
    FROM golang:1.20-alpine AS build: Эта строка определяет базовый образ для этапа сборки. 
    Он начинается с официального образа Golang Docker, помеченного как версия 1.20 и основанного на Alpine Linux. 
    Часть AS buildдает этому этапу имя "build", на которое вы можете сослаться позже.
WORKDIR /app: Эта строка устанавливает рабочий каталог внутри контейнера на /app. 
    Docker будет выполнять последующие команды в этом каталоге.
COPY . .: Эта команда копирует содержимое текущего каталога (предположительно, исходный код вашего приложения Go и другие необходимые
 файлы) в /appкаталог внутри контейнера.
RUN go build -o server ./cmd/server: Это команда, которая создает приложение Go. 
Она использует go buildкоманду для компиляции кода Go в текущем каталоге и вывода двоичного файла как 
    server. ./cmd/serverАргумент — это расположение кода приложения относительно /appкаталога.
  
Вторая часть
    FROM alpine:latest: На последнем этапе вы можете начать с базового образа Alpine Linux. 
    latestТег указывает на последнюю доступную версию Alpine Linux.
WORKDIR /app: Эта строка устанавливает рабочий каталог внутри контейнера на /app. 
Docker будет выполнять последующие команды в этом каталоге.
COPY --from=build /app/server .: Эта команда копирует двоичный файл с именем serverиз предыдущего 
    "этапа сборки" в /appкаталог внутри конечного контейнера. Этот двоичный файл — скомпилированное приложение Go, к
    оторое вы построили на этапе сборки.
EXPOSE 8080: Здесь вы указываете, что ваше приложение будет прослушивать порт 8080. 
    Это объявление, которое на самом деле не открывает порт; это способ задокументировать, 
    какой порт ваше приложение ожидает использовать.
CMD ["./server"]: Эта команда будет выполнена при запуске контейнера на основе образа. 
    Она указывает на запуск serverдвоичного файла, который является вашим приложением Go. 
    Эта команда запускает ваше приложение внутри контейнера.    
    
Создаем docker-compose.yml
        
Запускам сборку 
    docker build -t betterapp (это название приложения) . 
    
    docker-compose build
    docker-compose up -d 
   
   и после уже main.go
